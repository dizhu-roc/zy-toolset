# 密码生成器：设计结论与规划

本文档按《[新工具通用设计工作流](./新工具通用设计工作流.md)》的**四维度**梳理「随机强密码生成」工具：功能范围、页面结构、SEO 与说明文案；并简要对照常见竞品能力，给出可落地的设计结论。与站内路由 `tools/generator/password` 及 `messages` 中 `tools.passwordGenerator` 文案可对齐迭代。

---

## 一、功能范围

### 用户来这一页最想解决什么？

一次性拿到一串**足够随机、可按规则配置**的强密码，能马上复制去注册或改密；并清楚知道**是否仅在本地生成**（默认不上传服务端）。

### 功能清单

| 层级 | 能力 | 说明 |
|------|------|------|
| **必做** | 长度可调 | 建议范围如 8–64 或 12–128，默认 16–20；滑块 + 数字输入保持一致。 |
| **必做** | 字符集开关 | 大写 / 小写 / 数字 / 符号四类可勾选；至少一类为真，否则禁止生成并提示。 |
| **必做** | 再生成 | 每次用密码学安全随机源重新生成。 |
| **必做** | 复制 + 反馈 | 复制成功/失败提示，与站内其它工具交互习惯一致。 |
| **必做** | 随机源 | 使用 **`crypto.getRandomValues`**，不以 `Math.random()` 作为主熵源。 |
| **必做** | 本地说明 | 明确：仅在当前浏览器标签页内生成，默认不上传；链到站点隐私说明。 |
| **必做** | 约束满足 | 若提供「每类至少一个」选项，生成结果须满足（实现上可校验后重试或先铺样本再均匀打乱）。 |
| **可选** | 排除易混字符 | 如 `0`/`O`、`1`/`l`/`I` 等，单开关即可。 |
| **可选** | 符号集策略 | 「全符号」与「键盘常见安全符号」子集二选一，减少部分系统对特殊字符不兼容的问题。 |
| **可选** | 强度提示 | 按长度与字符空间粗算熵（比特）或弱/中/强标签即可，首版不必做密码语义级分析。 |
| **可选** | 清空 | 清空结果并将选项恢复默认。 |
| **后置** | 口令短语（Diceware） | 与「字符型随机密码」产品形态不同，建议**另页或另工具**，避免单页职责膨胀。 |
| **后置** | 批量多条 | 竞品部分支持；首版可不做。 |
| **不做** | 存储、同步、账号 | 不做迷你密码管理器。 |
| **不做** | 泄漏查询、登录检测 | 依赖第三方与合规说明，首版不碰。 |
| **不做** | 服务端生成密码 | 与「本地随机」卖点冲突，除非产品策略整体转向。 |

**谁在用：** 普通用户（注册网站）为主，开发者为辅（临时密钥）→ 文案少堆标准编号，多写随机、强度、复制、本地。

## 二、页面结构

密码工具的主「输入」是**规则**而非长文本，但仍可按「一侧控规则、一侧看结果」组织。

- **大屏：** 左（或上）规则区：长度、字符集、（可选）易混排除与符号策略、「每类至少一个」、再生成；右（或下）结果区：大号等宽展示当前密码、复制、（可选）一行强度说明。  
- **小屏：** 规则在上、结果在下；复制按钮靠近结果区，避免够不着。  
- **全站壳：** 仍使用 `ToolPageLayout`；交互放在独立 Client 子组件（与现有 Base64 专页模式一致）。  
- **合规与信任：** 页脚或 `ancillary` 保留隐私链接（当前占位页已有雏形）。

---

## 三、SEO 与关键词

- **价值一句话：** 在浏览器内生成可配置长度的随机强密码，默认不上传。  
- **关键词方向（心里或运营用）：** 随机密码、强密码、密码生成器、在线生成；若自然可加入「本地」「不保存」。  
- **与站内区分：** 不与 `base64/*`、文本编解码等页面抢同一批词；URL 保持 **`tools/generator/password`** 长期稳定。  
- **产出物：** `metaTitle` / `metaDescription` 与各语言 `messages` 键对齐迭代。

---

## 四、说明文案

- **页内短说明（建议 2 段以内）：** 是什么（按勾选规则在本地随机生成）、不能干什么（不保存、不同步、不查泄漏）、数据是否离开本机（默认不）。  
- **略长补充：** 可折叠一段，说明随机密码仍须自行妥善保存（如密码管理器）；不必首版就上长篇帮助中心。  
- **与通用工作流一致：** 若仅本地处理，必须在说明中写清，利于信任与搜索。

## 五、竞品能力简要对照

| 能力 | 常见程度 | 本产品建议 |
|------|-----------|------------|
| 长度 + 四类字符开关 | 几乎标配 | **必做** |
| 排除易混字符 | 很常见 | 首版可选或第二版 |
| 强调 Web Crypto / 客户端处理 | 常见 | **必做（实现 + 文案）** |
| 强度 / 熵展示 | 较常见 | 可选 |
| 符号集细分 | 部分有 | 可选 |
| 口令短语 / 多词密码 | 部分独立产品 | **另页**，不挤占本页 |
| 一次生成多条 | 少数 | 后置 |

---

## 六、设计结论（摘要）

1. **定位：** 轻量、本地、可复制的强随机密码生成器；不做存储与泄漏类服务。  
2. **MVP：** 长度 + 四类开关 + 至少一类校验 + `crypto.getRandomValues` + 再生成 + 复制 + 本地与隐私说明。  
3. **迭代顺序：** MVP → 易混排除 → 符号子集 → 熵/强度展示 →（再议）口令短语专页、批量生成。
