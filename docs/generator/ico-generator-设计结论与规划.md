# ICO / 网站图标生成器：需求与开发规划

本文档按《[新工具通用设计工作流](./新工具通用设计工作流.md)》的**四维度**梳理「从图片生成多尺寸图标与 ICO」工具；补充**需求规格**与**开发规格**，并对照常见竞品能力给出可落地的设计结论。实施时可与站内路由 `tools/generator/ico-generator`（草案）及 `messages` 中 `tools.icoGenerator` 文案对齐迭代。

---

## 一、功能范围

### 用户来这一页最想解决什么？

上传一张图，在**固定 1:1 取景框**内选好要露出的区域，选好要出的**像素尺寸**和**圆角**，在浏览器里生成 **ICO** 与/或**各尺寸位图**，打包 **ZIP 下载**，用于网站 favicon、PWA 或桌面快捷方式等；**不依赖服务端存图**。

### 功能清单

| 层级 | 能力 | 说明 |
|------|------|------|
| **必做** | 图片上传 | 支持常见栅格格式（建议首版：**PNG / JPEG / WebP**；GIF 动图仅首帧或明确不支持）。大小上限与内存提示（如 >10MB 警告）可配置。 |
| **必做** | 图片预览 + 裁剪 | 预览区**高度固定**；画布内**图片固定不动**（不拖图）；**仅裁剪框可拖动**（必要时允许缩放裁剪框边长，但比例恒为 **1:1**）。裁剪框不得超出图源有效区域（边界钳制）。 |
| **必做** | 输出尺寸多选 | **16×16、32×32、64×64、512×512** 四档可勾选；**默认勾选前三档**（16、32、64）。至少选一项才可生成/下载。 |
| **必做** | 圆角率 | **0 / 10 / 20 / 40 / 60 / 100（%）** 单选，**默认 0%**。语义建议：**圆角半径占「图标半边长」的比例**，100% 即为圆形图标。 |
| **必做** | ICO 生成 | 生成标准 **多图幅 ICO**（仅包含用户**已勾选**的尺寸；图幅建议用 **PNG 压缩**以保留透明，兼容 Vista 以来常见系统）。 |
| **必做** | 右侧信息区 | **ICO/图标预览**（见下文「预览策略」）、**当前文件名**、**已选尺寸列表**、**圆角率**；随左侧操作**实时更新**（含裁剪框移动、选项变更、重新上传）。 |
| **必做** | ZIP 下载 | **一个 ZIP**，内含**与勾选尺寸数量一致**的**栅格图片文件**（建议 **PNG**，文件名如 `icon-16x16.png` …）；**不含**多图幅单文件时的重复逻辑——即「选几个尺寸 = 几张图」。 |
| **必做** | 本地处理说明 | 明确图片仅在当前浏览器内处理，默认不上传服务端；链到站点隐私说明。 |
| **可选** | 单独下载 `.ico` | 与 ZIP 并列：一键仅下 **favicon.ico**（内含所选尺寸）。若产品希望「只有一个下载按钮」，可做成下拉：ZIP / ICO。 |
| **可选** | 裁剪框缩放 | 双指或角落手柄在 1:1 下缩放取景范围（竞品常见）；首版可仅平移裁剪框以控 scope。 |
| **可选** | 背景色 | 当图源有透明区域且用户希望导出非透明 ICO 时填充底色（首版可不做，文档占位）。 |
| **后置** | 更多预设尺寸 | 如 48×48、180×180（Apple Touch）等，与 PWA manifest snippet 可放迭代。 |
| **后置** | SVG 上传 | 光栅化路径与体积控制另议。 |
| **不做** | 服务端持久化、账号图库 | 与「本地处理」定位冲突。 |
| **不做** | 批量多文件队列 | 单页单图源即可，避免做成通用图像批处理。 |
| **不做** | `.icns`、完整品牌 VI 导出 | 非本站核心场景。 |

**谁在用：** 前端 / 全栈开发者、独立站长为主，偶尔有设计师 → 文案可提 favicon、多分辨率、ZIP、透明，避免过度 OS 内部格式细节。

**是否只在浏览器本地处理：** **是**；须在说明与 SEO 描述中写清。

## 二、页面结构

### 信息从哪进、从哪出？

- **输入：** 本地文件 + 裁剪几何 + 尺寸多选 + 圆角率。  
- **输出：** 右侧预览 + ZIP（N 张 PNG）+（可选）单个 ICO 下载。

### 线框级布局（区块名 + 大致布局）

| 区域 | 内容 |
|------|------|
| **左栏（控制区）** | ① 上传区（拖拽/点击）；② **固定高度**裁剪预览（图源铺满或 cover 后固定，**裁剪框** 1:1 可拖）；③ 尺寸四 Checkbox（默认前三项勾选）；④ 圆角率单选（6 档）；⑤ 主操作：**生成**（或实时生成，见开发说明）+ **下载 ZIP** +（可选）**下载 ICO**。 |
| **右栏（结果与摘要）** | ① **预览**：主预览可展示「当前最大已选尺寸」的缩略表现，或 16/32/64 多格小预览（实现取简）；② **已选信息**：文件名、尺寸列表（标签或列表）、圆角率；与左侧**状态绑定**，无缓存延迟感。 |
| **小屏** | 上：上传 + 裁剪；中：尺寸与圆角；下：预览与摘要 + 下载。 |
| **全站壳** | `ToolPageLayout`（Server）；裁剪、Canvas 导出、ZIP 在 **独立 Client 子组件** 中实现，与现有 Base64 / 密码生成器模式一致。 |

### 裁剪交互（需求细化）

- 预览容器：**固定高度**（如 `h-64`～`h-80` 量级，设计 token 与现网一致即可），宽度 `w-full`，内层 **图源等比 cover 居中**，**不可拖拽/缩放图源**（首版）。  
- **裁剪框**：正方形，在「图源在屏幕上的映射坐标」内平移；边界与图源可视区域求交，禁止露出空白（或允许露出时置灰——建议 **禁止露出空白**）。  
- 将「屏幕上的裁剪框」映射回**原始像素坐标**的 1:1 矩形，供下游缩放至各输出尺寸（保证 16 与 512 来自同一取景语义）。

## 三、SEO 与关键词

| 产出 | 建议草案 |
|------|-----------|
| **价值一句话** | 在浏览器内从图片裁剪 1:1 区域，生成多尺寸网站图标与 ICO，打包 ZIP 下载，图片不上传服务器。 |
| **title / description** | 含 *favicon*、*ICO*、*图标生成*、*多尺寸*、*ZIP*、*本地* 等自然组合；各语言 `messages` 与 `generateMetadata` 对齐。 |
| **URL** | 建议 slug **`tools/generator/ico-generator`**（与生成器产品线一致、长期稳定）。 |
| **站内区分** | 不与 Base64、纯密码生成器等抢词；与后续「纯 SVG 转 ICO」类页可并存，靠描述区分。 |

## 四、说明文案

- **页内短说明（2 段以内）：** 工具用途（favicon / 多尺寸 PNG + ICO）；裁剪与圆角含义；**数据不离开本机**（默认）。  
- **折叠补充（可选）：** ICO 在极老系统上的兼容说明；建议图源尽量 ≥512 边长以获得清晰 512 输出。  
- **错误态：** 未上传、未选尺寸、生成失败（内存、超大图）时简短可执行提示。

## 五、竞品能力简要对照

| 能力 | 常见程度 | 本产品建议 |
|------|-----------|------------|
| 上传 + 方形裁剪 | 几乎标配 | **必做**（固定 1:1；图不动框动） |
| 多尺寸勾选 / 预设集 | 很常见 | **必做**（四档 + 默认前三） |
| ZIP 多 PNG | 很常见 | **必做** |
| 单文件多分辨率 ICO | 很常见 | **必做** |
| 客户端处理、无上传 | 常见卖点 | **必做（实现 + 文案）** |
| 圆角 / 遮罩 | 部分有 | **必做**（按给定 6 档） |
| HTML / manifest 片段 | 较常见 | **后置** |
| SVG 优先工作流 | 部分站点 | **不做（首版）** |
| 文字 / Emoji 生成 favicon | 少数 | **不做**，避免单页膨胀 |

参考站点（仅能力对标，非背书）：[FaviconGenerator.io](https://favicongenerator.io/)、[favicongenerator 类工具聚合](https://kordu.tools/tools/image/favicon-generator/) 等普遍具备「上传 → 裁切 → 多尺寸 → ZIP/ICO → 本地处理」链路。

## 六、设计结论（摘要）

1. **定位：** 本地、单图、1:1 裁剪 → 多尺寸栅格 + 标准 ICO；下载以 **ZIP（N 张 PNG）** 为主交付物，ICO 为强关联能力（单独按钮或下载项）。  
2. **MVP：** 上传 + 固定高度预览 + 可拖裁剪框 + 四尺寸多选（默认三选）+ 圆角六选一 + 实时右侧摘要与预览 + ZIP + ICO。  
3. **迭代顺序：** MVP → 单独 ICO 按钮与命名规范 →（可选）裁剪框缩放 → 更多尺寸与 manifest 片段。

## 七、需求规格（验收向）

### 7.1 功能需求

| ID | 描述 | 验收要点 |
|----|------|-----------|
| FR-1 | 用户可选择本地栅格图片上传 | 更换文件后裁剪状态重置或按产品约定重置取景 |
| FR-2 | 裁剪区高度固定，比例 1:1 | 仅裁剪框移动；图源在预览内固定；导出各尺寸内容一致 |
| FR-3 | 尺寸 16/32/64/512 多选 | 默认 16、32、64 勾选；全不选时禁用生成/下载并提示 |
| FR-4 | 圆角 0/10/20/40/60/100% 单选 | 默认 0%；切换后预览与导出一致 |
| FR-5 | 生成结果 | 所选每个尺寸各一张 PNG（逻辑分辨率正确）；圆角应用于整张方图 |
| FR-6 | ICO | 内含且仅内含所选尺寸的图幅；透明通道保留 |
| FR-7 | ZIP | 内含文件数 = 勾选尺寸数；扩展名与 MIME 正确 |
| FR-8 | 右栏 | 展示预览 + 文件名 + 尺寸列表 + 圆角；与左栏状态同步 |

### 7.2 非功能需求

| ID | 描述 |
|----|------|
| NFR-1 | 全程无默认服务端图像上传；网络请求仅站点静态资源与可选分析 |
| NFR-2 | 常见笔记本上 2048 以内边长图源可交互裁剪；超大图降级策略（缩放采样或提示）需在实现中明确 |
| NFR-3 | 无障碍：上传与勾选有可读标签；键盘可操作勾选（尽量） |

### 7.3 右侧「ICO 预览」策略（实现二选一或组合）

- **方案 A（推荐 MVP）：** 以 **最大已选尺寸** 的生成结果缩略显示（单图），下附文字「预览按最大尺寸缩放」。  
- **方案 B：** 16/32/64 三格微型预览（512 过大可省略格或单独一行）。  

文档要求：**至少一种**随参数实时更新的视觉反馈即可通过验收。

## 八、开发规格

### 8.1 路由与工程对齐

- 页面：`src/app/[locale]/tools/generator/ico-generator/page.tsx`（slug 以最终实现为准，与 `hrefForLocale` 一致）。  
- 文案：`src/messages/*.json` 增加 `tools.icoGenerator`（与 `getMessages` 类型一致）。  
- `generateMetadata`：`title` / `description` / `robots: { index: true, follow: true }`。  
- `src/app/sitemap.ts` 纳入 `tools/generator/ico-generator`。  
- 首页或工具矩阵入口：按《[工具规划](../工具规划-开发类工具产品线.md)》与运营节奏添加。

### 8.2 技术要点

| 模块 | 建议 |
|------|------|
| 裁剪与坐标 | 维护 `sourceImage` 自然尺寸与预览 DOM 尺寸比例；裁剪矩形 `(sx, sy, sSize)` 全为源像素整数，避免累积误差 |
| 圆角 | 离屏 `canvas`：`roundRect` + `clip` 后 `drawImage`；或先方图再蒙版；100% 为圆 |
| 缩放 | `drawImage` 至 `size × size`；长边采样建议 `imageSmoothingQuality = 'high'` |
| ICO | 使用成熟轻量库将多帧 PNG 合成 ICO，或按规范手写 DIR + BITMAPINFO（优先库以减坑） |
| ZIP | `JSZip` + `file-saver` 或浏览器 `showSaveFilePicker` 降级策略与现网其它下载一致 |
| 性能 | 大图 debounce 预览；生成可显式按钮触发以免每像素移动都打全量 |

### 8.3 组件拆分（建议）

- `IcoGeneratorPanel`（Client）：状态机（未上传 / 已上传 / 生成中 / 错误）。  
- `IcoCropStage`：固定高度容器 + overlay 裁剪框拖拽。  
- `IcoOutputSidebar`：预览 + 元信息 + 下载区。  
- `lib/ico-export.ts`（或类似）：从裁剪参数 + 圆角 + 尺寸列表 → `Blob[]` / ICO `Blob`。

### 8.4 依赖与体积

- 评估 `jszip`、ICO 相关包的 gzip 体积；必要时动态 `import()` 在首次点击下载时加载。

### 8.5 测试建议

- 单测：裁剪坐标钳制、圆角 0 与 100% 像素抽样、ICO 所含尺寸种类与顺序。  
- 手测：透明 PNG、不透明 JPEG、仅选 512、四尺寸全选、反复改裁剪后 ZIP 文件数与内容。
