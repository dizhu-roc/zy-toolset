# 新工具通用设计工作流（四维度）

本文档为**全站新工具**可用的通用检查表，与具体业务（Base64、密码生成等）无关。此前部分内容位于《[文本 Base64 编码：设计结论与工作流](../base64/文本Base64编码-设计结论与工作流.md)》第一节，已抽离至此，便于各产品线文档统一引用。

## 使用方式

每个新工具建议按下面顺序想一遍（顺序可微调，不建议跳步）。把下表当**检查表**，开干前先填一遍，可减少返工。

| 步骤 | 要想清楚的事 | 建议产出 |
|------|----------------|----------|
| **1. 功能范围** | 用户来这一页最想解决什么？次要需求有哪些？哪些**明确不做**（避免做成万能箱）？ | 功能清单：必做 / 可选 / 不做 |
| **2. 页面结构** | 信息从哪进、从哪出？要不要对比、要不要分步？大屏是否**左右分栏**（输入一侧、输出一侧）？小屏如何上下叠放？ | 线框级结构：区块名 + 大致布局 |
| **3. SEO 与关键词** | 用户会怎么搜？标题和描述能否**一句话说清价值**？是否和站内其它页面抢同一批词？URL 是否长期稳定？ | `title` / `description` 草稿 + 若干自然说法的关键词 |
| **4. 说明文案** | 要不要解释「是什么、能干什么、不能干什么、数据是否上传」？写多长、放在哪（页内折叠 / 链到帮助页）？ | 是否需要说明段 + 放置方式 |

**补充两点（建议挂在步骤 1 前后）：**

- **谁在用**（普通人 / 开发）→ 决定用词深浅。  
- **是否只在浏览器本地处理** → 若属实，在说明里写清楚，有利于信任与长尾搜索。
