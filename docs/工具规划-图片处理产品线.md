# 工具规划：图片处理产品线

> **范围**：浏览器内 **Canvas / OffscreenCanvas / Bitmap、decodeImageBitmap、编码 API（mozjpeg 等 WASM 可选）** 能覆盖的常见图片工具；不含必须服务端的大图流水线。  
> **原则**：纯前端优先；大文件需**上限与降级策略**（降采样、拒绝、Web Worker）。  
> **状态**：规划文档；**不**代表已实现；与站内已有 `image/compress`、`image/resize` 等可对齐或合并（见 §5）。

---

## 1. 分类框架

| 大类 | 用户意图 | 纯前端可行性 |
| --- | --- | --- |
| 体积与编码 | 压缩、格式转换 | 高；质量与速度依赖实现（WASM 可选）。 |
| 几何 | 缩放、裁剪、旋转、翻转 | 中高；极大尺寸受内存限制。 |
| 元数据 | EXIF 查看/剥离 | 中；需解析 JPEG/TIFF 等；剥离后重编码。 |
| 合成与导出 | 拼图、水印、圆角、阴影 | 中；逻辑复杂度主要在交互。 |
| 文档化输出 | 转 PDF、多图打包 | PDF 已有方向；ZIP 纯前端可行。 |
| 专业向 | ICC 校色、RAW | 低～中；RAW 体积与解码库限制大，可作为远期。 |

## 2. 工具清单（建议 slug · 优先级）

### 2.1 P0（高频、与现有站高度重合或应统一品牌）

| 规划名 | 建议 slug | 说明 |
| --- | --- | --- |
| 图片压缩 | `image/compress` | 控制质量 / 长边；JPEG、WebP、PNG 策略不同。 |
| 尺寸调整 | `image/resize` | 按宽高或长边；可选放大策略（插值说明）。 |
| 格式转换 | `image-format-converter` | 单图转 JPEG / WebP / PNG；可与压缩同组或同页 Tab。 |
| 多图 → PDF | `image-pdf-generator` | A4 排版；多页导出等按产品需要扩展。 |
| ICO / 多尺寸 PNG | `generator/ico` | 已有方向；与 favicon 场景一致。 |

### 2.2 P1（强需求、纯前端可做）

| 规划名 | 建议 slug | 说明 |
| --- | --- | --- |
| 图片裁剪 | `image/crop` | 固定比例 / 自由裁剪；导出为 Blob。 |
| 旋转与翻转 | `image-rotate-flip` | 90° 旋转、水平垂直翻转；可并入裁剪页 Tab。 |
| 圆角 / 简单圆角导出 | `image-rounded-corners` | Canvas 圆角矩形裁剪；或与裁剪合并。 |
| EXIF 查看器 | `image-exif-viewer` | 只读展示；隐私说明（地理位置等）。 |
| 去除 EXIF | `image-strip-metadata` | 重编码输出；与压缩可共用导出。 |
| 图片水印（文字 / 图片） | `image-watermark` | 透明度、平铺、单角；注意字体授权若用 Web 字体。 |
| 图片拼接（横向 / 纵向） | `image-stitch` | 多张等高或等宽拼接；画布尺寸上限。 |
| 九宫格 / 简单切片 | `image-split-grid` | 均分切片下载 ZIP。 |
| 颜色拾取（从图取色） | `image-color-picker` 或独立 `color-picker` | 画布取像素 RGB/HEX。 |

### 2.3 P2（有场景、实现或 SEO 成本较高）

| 规划名 | 建议 slug | 说明 |
| --- | --- | --- |
| 图片模糊 / 像素化（打码） | `image-pixelate` | 隐私打码；固定区域或全图。 |
| 亮度 / 对比度 / 饱和度 | `image-adjustments` | 简单调色；与「设计向」工具区分。 |
| 透明背景（简单抠图） | `image-remove-background` | **纯前端**可用色域阈值 / 魔棒，效果有限；若宣传「AI 抠图」则需服务端模型。 |
| GIF 拆帧 / 合成简易 GIF | `gif-frame-splitter` / `gif-maker` | 帧数与体积限制；解码库体积。 |
| SVG 栅格化 | `svg-to-png` | 内联 SVG → Canvas → PNG；复杂 SVG 兼容性需说明。 |
| 渐进式 JPEG / 自适应质量 | 并入压缩工具选项 | 不必独立 slug。 |
| WebP 动图 / AVIF | 视浏览器解码支持 | AVIF 编码浏览器支持度需查；规划为可选。 |

### 2.4 批量与打包（P1～P2）

| 规划名 | 建议 slug | 说明 |
| --- | --- | --- |
| 批量压缩 / 转换 | `image-batch-converter` | 多文件队列 + ZIP 下载；内存与耗时提示。 |
| 统一长边 / 统一宽高 | 可并入 batch 或 resize | 减少 slug 数量。 |

## 3. SEO 与 slug 策略

- **优先**：少数 **强落地页**（压缩、缩放、格式、PDF、裁剪）+ 清晰标题与独立说明段落。  
- **合并**：旋转、圆角、简单调色可作为 **「图片编辑器」单页多 Tab**，避免十个薄页互抢「在线 图片 工具」。  
- **谨慎独立 slug**：`remove-background` 等易被期望为「一键 AI」，若纯前端效果弱，反而伤体验与口碑。

## 4. 技术备注（落地前评估）

| 项 | 说明 |
| --- | --- |
| 色彩空间 | Canvas 默认 sRGB；打印 ICC 几乎不碰。 |
| 透明 | PNG / WebP 保留 Alpha；JPEG 无透明需铺底说明。 |
| Worker | 大图编码放 Worker 避免卡 UI；需传递 ImageBitmap。 |
| HEIC | 浏览器解码能力有限；若做需列支持矩阵或依赖 WASM。 |
| 安全 | 不上传；仍须在文案中说明剪贴板 / 本地文件范围。 |

## 5. 与站内已有能力对齐（仅对照，不要求本文档即改代码）

以下在仓库中已有独立工具方向（名称以 slug 为准）：`image/compress`、`image/crop`、`image/resize`、`image-format-converter`、`image-pdf-generator`、`generator/ico`。  
规划上：**P0 新需求优先并入上述页的 Tab 或子模块**，再考虑新增 slug，避免重复入口与 canonical 负担。

---

## 6. 建议路线图（产品）

1. **统一「图片工具」信息架构**：Hub 或工作台式入口，列出 P0～P1。  
2. **先做裁剪 + 旋转**（与缩放、格式形成闭环）。  
3. **EXIF 查看 / 剥离**（隐私敏感场景清晰文案）。  
4. **批量 + ZIP**（与单图能力复用编码管线）。  
5. **评估** GIF、SVG、AVIF 与「抠图」是否值得独立品牌页。
