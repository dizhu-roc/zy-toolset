# 工具规划：生成器产品线

> **范围**：面向「**一次性产出可配置或可复制的产物**」的工具族——不依赖用户已有输入体做「变换」，而是**由参数 / 规则 / 熵**生成结果。本文将你点名的 **密码、UUID、Sitemap、cURL、Crontab、二维码** 视为**同一产品簇**（竞品常统称 *Generators* 或 *Developer utilities* 下的一块）。  
> **原则**：能纯前端完成的优先本地执行；涉及随机性的需说明**熵来源**（`crypto.getRandomValues` 等）；站点类产物（sitemap）需标明**导出格式与体积建议**。  
> **状态**：规划文档；**不**绑定当前仓库路由实现。

## 2. 能力族谱（为何能归为一类）

本簇工具在**工程上**并不共享同一套算法，但在**产品上**共享同一用户命题：「**我现在要造一个可用的东西出来（字符串 / 表达式 / 文件内容 / 图）**」。

| 子族 | 代表工具 | 共性 | 典型输出 |
| --- | --- | --- | --- |
| **随机与熵** | 密码生成器、UUID | 不可预测或可配置随机性；需文案强调**本地、不上传** | 字符串、多行批量 |
| **标识与时间调度** | Crontab 生成器、（扩展）时间戳生成 | 规则 → 机器可读表达式；人类可读反向解释可增强信任 | Cron 表达式、说明文字 |
| **可视化载体** | 二维码生成器 | 编码 + 渲染；导出 PNG/SVG；容错级别 | 图片或矢量 |
| **请求与集成草稿** | cURL 生成器 | 表单 / 方法 / Header → 命令行；与 API 调试文档强相关 | 可复制的 shell 片段 |
| **站点与爬虫制品** | Sitemap 生成器 | URL 列表 → `urlset` XML；与 SEO、静态托管流程相关 | XML 文本或下载 |

**边界（可与生成器同 Hub、文档上可拆子类）**：Lorem、随机 Hex、JWT 组装等——竞品常与 UUID/密码同列；哈希**摘要**更偏「密码学运算 / 校验」，可单独成工具页，不必与本文簇共用 slug。

| 策略 | 说明 |

## 4. 你点名的工具 → 规划映射

| 规划名 | 建议 slug（草案） | 归入子族 | 备注 |
| --- | --- | --- | --- |
| 密码生成器 | `generator/password` | 随机与熵 | 长度、字符集、可读性选项；禁止默认把密码写入 URL |
| UUID 生成器 | `uuid-generator` | 随机与熵 | v4 为主流；v1 与时间/隐私说明 |
| 二维码生成器 | `qr-code-generator` | 可视化载体 | 容错、尺寸、内容类型（URL / Wi‑Fi / vCard 等可分期） |
| Crontab 生成器 | `crontab-generator` | 标识与时间调度 | 五段 UI + 人类可读解释 + 常见预设 |
| cURL 命令生成 | `curl-generator` | 请求与集成草稿 | 与「仅浏览器内发请求」类工具区分：此处**产出命令**即可纯前端 |
| Sitemap 生成 | `sitemap-generator` | 站点与爬虫制品 | 根 URL、路径列表、hreflang 可选；大列表性能说明 |

以上六项在竞品导航中**高频共现**，适合在站内用**同一品牌区块**（首页一块、顶栏一项、Hub 一页）承接。

## 5. 验收与文案要点

- 随机类：默认 **本地生成**、不记录服务端（若有埋点需合规）。  
- Crontab：时区与「五段 vs 六段」说明。  
- cURL：提示 **shell 转义**、Windows `curl.exe` 差异（可在页面内附短文说明）。  
- Sitemap：`xmlns`、条数上限、与 robots 的关系（简述即可）。  
- 二维码：长文本 / 大容量内容的**版本与扫描距离**说明。
