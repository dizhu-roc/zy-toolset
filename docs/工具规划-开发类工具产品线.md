# 工具规划：开发类工具产品线

> **范围**：面向日常 **联调、排障、读日志、读浏览器信息** 的**纯前端**小工具簇。本文将下列五项归为同一产品类 **「开发类工具」**（工作名，对外可称 *Developer utilities* / *Dev tools*）：  
> **HTTP Header / Cookie 解析**、**User-Agent 分析**、**cURL 分解**、**时间戳 / ISO / 时区转换**、**颜色转换器**。  
> **原则**：输入默认**仅在浏览器内处理**；涉及 Cookie、Authorization、UA 等时须显著 **隐私提示**；时间与时区须写清 **数据源（本机 Intl / 打包的 IANA 数据）** 与 **DST 边界**。  
> **状态**：规划文档；**不**绑定当前仓库路由。

---

## 1. 为何归为一类

| 维度 | 说明 |
| --- | --- |
| **用户命题** | 「我有一段**从网络、日志、文档里复制来的东西**，想**看懂结构**或**换成另一种等价表示**。」 |
| **与「编码类」区别** | 编码类侧重 **A↔B 字符表示**（Base64、URL 等）；本文簇侧重 **协议面、运行环境面、时间与颜色空间** 的**解析与换算**。 |
| **与「生成器」区别** | 生成器侧重 **从参数造新物**；本文簇侧重 **解析已有文本** 或 **在已知标量间转换**（时间、颜色）。 |
| **竞品习惯** | DevTools 套件、IT-Tools 类站点常把 Header、JWT、时间戳、颜色等放在**同一「开发者工具」导航**下，便于发现。 |

---

## 2. 能力族谱（子类）

| 子类 | 工具 | 输入 / 输出形态 | 纯前端要点 |
| --- | --- | --- | --- |
| **协议文本解析** | HTTP Header + Cookie 解析 | 原始 `Request` 头块或单行 `Cookie` | 多行合并、重复键、`Set-Cookie` 多枚拆分；**不**自动附带真实请求 |
| **运行环境指纹** | User-Agent 分析 | UA 字符串 + 可选 Client Hints | 规则表可本地打包或轻量解析；**勿**把完整 Cookie 与 UA 一并上报第三方 |
| **命令行互操作** | cURL 分解 | 一条或多条 `curl ...` | 覆盖常用 flags（`-H` `-d` `-X` `--data-binary` 等）；无法解析时**降级为原文分区**而非静默错 |
| **时间与历法** | 时间戳 / ISO / 时区 | Unix 秒/毫秒、ISO8601、选时区 | 毫秒与秒误识别提示；时区库体积与**默认 IANA 区名**；Locale 展示可选 |
| **颜色空间** | 颜色转换器 | HEX / RGB / HSL / OKLCH 等 | 解析容错、`#RGB` 简写；若含 **取色器**（从屏幕取色）可与转换器 **同页 Tab**，仍属本簇 |

## 3. 工具清单（建议 slug 与合并关系）

### 3.1 首推（P0）

| 规划名 | 建议 slug（草案） | 覆盖场景 | 备注 |
| --- | --- | --- | --- |
| HTTP Header / Cookie 解析 | `header-parser` 或 `http-header-parser` | 粘贴完整头、`Cookie` / `Set-Cookie`、`Authorization` 结构高亮 | 敏感头字段可折叠复制 |
| User-Agent 分析 | `user-agent-analyzer` | 浏览器、引擎、设备、架构；可选 CH | 可与 Header 解析同 Hub 并列展示 |
| cURL 分解 | `curl-parser` 或 `curl-command-parser` | 从文档/日志还原方法、URL、Header、Body | 与站内 cURL 生成类工具用户任务不同（读命令 vs 写命令） |
| 时间戳 / ISO / 时区 | `timestamp-converter` 或 `epoch-timezone-converter` | Unix ↔ 本地 ↔ 指定 IANA；相对「现在」 | 日志排障场景可在文案中说明 |
| 颜色转换器 | `color-converter` | HEX ↔ RGB ↔ HSL；可选 OKLCH | 取色可与转换器同页 Tab 或独立 `color-picker` slug |

### 3.2 可选扩展（P1）

| 规划名 | 建议 slug（草案） | 说明 |
| --- | --- | --- |
| 合并「开发类」Hub | `dev-utilities` / `developer-tools` | 聚合卡片 + 锚点；子页仍独立 slug 利于 SEO |
| Link 头 / CSP 简单拆解 | 并入 Header 解析 Tab | 专用解析器小、价值集中 |
| 常用 `Authorization` 子格式 | Bearer 展示、Basic Base64 **只解码展示**（风险提示） | Basic 解码属 Base64 字节语义，须在 UI 标明风险 |

### 3.3 刻意不包含（避免与本文簇混淆）

| 项 | 说明 |
| --- | --- |
| **真实 API 调试 / Webhook 收件** | 受 CORS 与公网入站限制；不归入「纯前端开发类」核心承诺。 |

## 4. SEO 与信息架构

| 策略 | 说明 |
| --- | --- |
| **Hub + 子页** | 首页或顶栏一个 **「开发类工具」**入口；子工具各保留 slug，避免单页堆砌导致首屏过长。 |
| **相关工具相邻** | cURL **生成**（造命令）与 **分解**（读命令）在导航或 Hub 上可相邻摆放，便于用户切换任务。 |
| **薄内容风险** | UA 分析页需有**可更新**的规则说明段落（版本、CH 字段表节选），避免全文仅一个输入框。 |

---

## 5. 验收与文案要点

- **Header / Cookie**：说明是否合并多行 continuation；`Set-Cookie` 属性（`HttpOnly` `Secure` `SameSite`）可读列表。  
- **UA**：说明规则来源与更新节奏；CH 需 **HTTPS / 权限** 的浏览器限制。  
- **cURL**：列出支持的 flags 子集；失败时给出**可编辑的手工分区**（URL 区、Header 区）。  
- **时间**：秒/毫秒切换显眼；**时区缩写**（如 `CST`）多义时在 UI 中优先 **IANA ID**。  
- **颜色**：超出 sRGB 的表示写清截断或拒绝策略；打印色若不做则明确「屏幕色」。  
- **全局**：首屏或页脚 **默认本地处理、不上传**。
