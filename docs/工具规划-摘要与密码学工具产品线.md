# 工具规划：摘要与密码学（Hash / HMAC / AES / CRC 等）

> **范围**：浏览器内可完成的 **摘要（Hash）**、**消息认证码（HMAC）**、**对称加密（以 AES 为主）**、**循环冗余校验（CRC）** 及与之一致的 **口令哈希（bcrypt 等）** 产品形态；可扩展 **编码输出**（Hex / Base64）、**文件与文本**输入。  
> **原则**：默认 **本地计算、不上传**；涉及密钥、IV、口令的页面须有 **风险提示** 与 **勿在 URL 传密** 的说明。  
> **状态**：规划文档；**不**代表已实现；与站内已有 `md5-hash-calculator`、`sha-hash-calculator`、`text-hash-calculator`、`hmac-hash-calculator`、`bcrypt-hash-calculator`、`aes-cipher-calculator`、`crc-calculator`、`hash-tools` Hub 等可对齐（见 §5）。

---

## 1. 能力与边界（纯前端）

| 类别 | 典型算法 | 浏览器侧要点 |
| --- | --- | --- |
| 摘要 Hash | MD5、SHA-1/2/3 | 文本 UTF-8、文件二进制；输出 Hex 为主，可选 Base64。 |
| HMAC | HMAC-SHA256 等 | 密钥与消息编码约定一致；侧信道不在 Web 场景重点。 |
| 对称加密 | AES（ECB/CBC/CTR 等） | 已有 CryptoJS 等路径；ECB 弱模式须在 UI 标注。 |
| CRC | CRC-8/16/32、多项式 | 非密码学摘要，用于完整性快速校验。 |
| 口令哈希 | bcrypt、可选 scrypt/Argon2 | bcryptjs 常见；Argon2 需 WASM 库，体积与兼容性单独评估。 |
| 非对称 | RSA / Ed25519 | **可选远期**：纯前端可做加解密与签名演示，密钥与 PEM 解析复杂度高。 |

**不推荐**在首屏承诺「军用级」「绝对安全」；应写清算法用途与误用场景（如 MD5 不应用于密码存储）。

## 4. 统一体验要点（落地时）

- **输入**：明文 UTF-8、Hex、Base64、文件；**编码**与算法声明一致。  
- **输出**：Hex 大小写、Base64、复制、下载小结果文件。  
- **密钥**：粘贴区默认不写入 URL；localStorage 若记忆密钥须单独同意与清除。  
- **对比模式**：两段 Hex 是否相等（摘要比对）可作为小功能，不必独立 slug。  
- **大文件**：Worker + 分块；失败重试与上限提示。
