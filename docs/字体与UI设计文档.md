# 在线工具集网站 — 字体与 UI 设计文档

> 文档用途：统一视觉与排版规则，保证多工具页面一致性与可访问性。  
> 实施时以代码中的设计 token（CSS 变量或主题配置）为准，本文描述目标态。

---

## 1. 设计原则

1. **内容优先**：大输入区、清晰主按钮；装饰克制，减少认知噪音。  
2. **工具站心智**：偏中性、偏「开发者工具」气质，避免过度营销风。  
3. **状态可见**：加载、成功、错误、禁用态样式统一，错误信息可读、可行动（如何修正）。  
4. **国际化友好**：布局预留文案变长空间（德语等未来扩展）；首版 en/zh 字符集与行高已考虑中文密度。  
5. **明暗可选（建议 P1）**：首版可仅亮色；若上暗色，同一套 token 映射即可。

---

## 2. 字体设计

### 2.1 字体栈（建议）

| 用途 | 西文 / 数字 | 中文 |
| --- | --- | --- |
| **界面与正文** | `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` | 在系统栈后追加 `"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif`（部署环境可改为 `Noto Sans SC` 网络字体子集以统一跨平台） |
| **等宽（代码、JSON、Base64、终端输出）** | `ui-monospace, SFMono-Regular, "SF Mono", Menlo, Consolas, "Liberation Mono", monospace` | 中文等宽回退同上无衬线即可；代码区以英文等宽为主 |

### 2.2 字号与行高（桌面基准，移动端 +1~2px 可调）

| Token | 用途 | 字号 | 行高 |
| --- | --- | --- | --- |
| `text-xs` | 辅助标签、元信息 | 12px | 1.4 |
| `text-sm` | 表单说明、页脚、表格次要列 | 14px | 1.5 |
| `text-base` | 正文、输入框内文字 | 16px | 1.55 |
| `text-lg` | 小节标题 | 18px | 1.45 |
| `text-xl` | 页内工具标题 | 20px | 1.35 |
| `text-2xl` | 页面 H1 | 24–28px | 1.25 |

代码/日志区：`text-sm` 或 `text-base` + `line-height: 1.5`，**禁止小于 12px** 的可读文案。

### 2.3 字重与排版

- 标题：`font-weight: 600`；正文：`400`；强调：`500` 或 `600`，少用全大写长句。  
- 中文段落：**适当字间距正常**即可；避免中西文混排时英文被强制拉伸。  
- 数字与单位：尽量使用等宽区或 `font-variant-numeric: tabular-nums` 于表格列（可选）。

## 3. 色彩系统（建议 token）

以下为语义化命名，具体 HEX 在实现时填入主题文件。

| Token | 用途 |
| --- | --- |
| `--color-bg` | 页面背景 |
| `--color-surface` | 卡片、面板背景 |
| `--color-border` | 默认边框 |
| `--color-text` | 主文案 |
| `--color-text-muted` | 次要说明 |
| `--color-primary` | 主操作、链接强调 |
| `--color-primary-hover` | 主操作悬停 |
| `--color-danger` | 错误、破坏性操作 |
| `--color-success` | 成功提示 |
| `--color-warning` | 警告（大文件、弱算法等） |

**对比度**：主文案对背景建议 ≥ 4.5:1；大号标题可适当放宽但保持可读。  
**语义色**：不仅依赖颜色传达状态（配合图标或文案）。

## 4. 间距与布局

- **基准网格**：4px；常用间距 8 / 12 / 16 / 24 / 32。  
- **最大内容宽**：阅读型说明 `max-width: 65ch`～`72ch`（`max-w-prose`）；**顶栏 / 主内容列 / 页脚** 与工具页外壳统一为 **`--measure-content`（80rem，即 1280px @ 16px）**，实现为 `max-w-content`；工具内表单/编辑器可再套 `max-w-prose` 或双栏以免单行过长。  
- **页边距**：移动端 16px；桌面 24–32px。  
- **区块间距**：相邻大区块 ≥ 24px。

## 5. 组件与模式

### 5.1 页面结构（工具页模板）

1. 顶栏：Logo、主导航、**语言切换**、（可选）搜索。  
2. 面包屑（可选）：利于 SEO 与深层返回。  
3. 标题区：H1 + 一句话说明 + 可选「本地处理」徽章/文案。  
4. 主工作区：输入 | 输出 分栏或上下栈叠；移动端单栏垂直流。  
5. 次要区：FAQ、限制说明、算法/时区数据源说明（按工具类型选填）。  
6. 页脚：版权、隐私、联系方式。

### 5.2 表单与输入

- 文本域：默认可垂直拉伸；最小高度 ≥ 120px（可按工具调整）。  
- 主按钮：**每个视口焦点区域一个最明显主操作**（格式化、转换、生成等）。  
- 次要操作为幽灵按钮或描边按钮，避免与主按钮视觉竞争。

### 5.3 代码与结果展示

- 使用等宽字体容器；长内容 **横向滚动** 或 **折行策略二选一**（JSON 建议缩进 + 必要时横向滚动）。  
- 提供「复制」按钮；复制成功给予轻量反馈（toast 或内联文案）。

### 5.4 表格与列表

- 斑马纹可选；表头 `font-weight: 600`。  
- 窄屏：卡片式堆叠或允许横向滚动并提示。

---

## 6. 图标与插画

- 首版推荐 **单色线性图标**（与中性 UI 一致），尺寸 20–24px 栅格对齐。  
- 避免大面积插画拖慢首屏；若使用 SVG，控制体积与内联/外链策略。

---

## 7. 动效与反馈

- 过渡时长：150–250ms 为宜；尊重 `prefers-reduced-motion` 减弱动效。  
- 加载：大计算用 **非阻塞** 指示（Spinner/Skeleton + 文案「处理中」）。

---

## 8. 国际化（与 UI 相关）

- 为翻译预留按钮最小宽度；避免把句子硬拆成多段无上下文拼接。  
- 日期/数字格式跟随 **当前语言环境**（`Intl`），与 PRD 中 en/zh 一致。
