# 文件转 Base64：按通用工作流梳理

本文按《[文本Base64编码-设计结论与工作流.md](./文本Base64编码-设计结论与工作流.md)》中的**四维度通用工作流**整理「文件 → Base64」能力：功能范围、页面结构、SEO 与说明文案。可与站内路由 `tools/base64/file-encode`（`Base64Workbench` 默认 `file` Tab）及实现代码对照阅读。

## 0. 补充两问（与文本工具一致）

| 问题 | 结论（文件场景） |
|------|------------------|
| **谁在用** | 开发者（嵌入 Data URL、配置片段、调接口传二进制）、偶尔也有「把某文件内容变成一串字符」的普通人。用界面文案时偏「操作说明」，原理可放折叠。 |
| **是否只在浏览器本地处理** | 是。读文件、编码、复制均在本机完成，不上传服务器；应在页内显眼说明。 |

## 1. 功能范围

### 1.1 用户来这一页最想解决什么？

**核心：** 选择**本地任意类型文件**（在约定大小内）→ 得到 **Base64 字符串** 或 常见的 **Data URL**（`data:…;base64,…`）→ 能**一键复制**到别处用。

与「纯文本打进去再编码」的区别：这里按**文件原始字节**编码，不经过「先当 UTF-8 文本再转字节」的路径；因此适合图片、PDF、小压缩包、任意二进制，而不仅限于文本文档。

### 1.2 功能清单：必做 / 可选 / 不做

| 类别 | 内容 |
|------|------|
| **必做** | 选本地单文件；按文件字节做 Base64；可选输出 **仅 Base64** 或 **完整 Data URL**（MIME 由浏览器 `File.type` 等推断，常见类型能对齐）；**编码/生成结果**、**复制结果**、**错误与上限提示**（如体积超限、读盘失败）。 |
| **必做（信任）** | 简短说明「本地处理、不上传」+ 可链到隐私政策（与现网 `ToolPageLayout` / workbench 一致即可）。 |
| **可选** | 展示已选**文件名、大小**；**清空/重新选择**以恢复初始状态；成功复制后的**轻量反馈**；对超大结果只做展示、提示「体积大可能卡顿」类文案（不强行替用户拆策略）。 |
| **建议刻意不做（本页）** | **Base64 解码**、**把编码当加密**的长篇教育（可一句话 + 链到别页/折叠）；**批量多文件**、**分片/流式**协议；**在浏览器里改 MIME 为任意自定义**（若要做，可列为远期）；与「文本转 Base64」页重复的长篇教学。 |
| **与 `text-encode` 的边界** | 文本页：多数字符串以 UTF-8 成字节再 Base64。文件页：**文件即字节流**，不强调「你输入的是一段 UTF-8 文本」。两页**关键词与标题**需区分，避免 SEO 上被视为重复。 |

### 1.3 与当前实现（`Base64Workbench` 文件 Tab）的对应关系

- **体积上限**：当前为 **8MB**（常量 `MAX_FILE_BYTES`），超限应提示并**不**强行编码。  
- **Data URL 开关**：勾选时用 `readAsDataURL`；不勾选时 `arrayBuffer` + `bytesToBase64`（纯 Base64）。  
- **本页可演进点**（未强行定案）：独立 `file-encode` 专页若追求与「文本」**左右分栏**一致，可把**选文件 + 元信息**与**只读输出区**在宽屏上做两列；**清空**、**与解码 Tab 的互链**可按信息架构再拍板。

## 2. 页面结构（线框级）

**信息走向：** 本地文件 →（可选选项）→ 只读大段输出 → 复制。

### 2.1 全页骨架（在 `ToolPageLayout` 内）

1. **标题 + 一句描述**（Server：`generateMetadata` 与首屏可见文案一致）。  
2. **合规/信任短链**（如隐私，与现网一致）。  
3. **主内容区**（见下）。

### 2.2 主内容区建议区块（与文本工具「左右分栏」精神对齐）

| 区域 | 宽屏 | 窄屏 | 内容要点 |
|------|------|------|----------|
| **信任与说明** | 通栏，短 | 同左 | 本地处理、非加密、用途一句话；可折叠「Data URL 与纯 Base64 差在哪」。 |
| **输入侧** | 左栏或上区 | 上 | 文件选择、已选文件信息、体积提示、错误信息（`role="alert"`）。 |
| **输出侧** | 右栏或下区 | 下 | 只读 `textarea` 或等宽区展示结果；**复制**以主按钮或次按钮出现。 |
| **选项** | 紧挨输入或两栏之间 | 输入与输出之间 | 是否生成 **Data URL**（与现实现一致）。 |
| **操作** | 输入/输出附近通栏 | 下 | **编码/生成**（若需显式按钮；或选文件后自动编——产品二选一）、**复制**、**清空**。 |

**与多 Tab 工作台关系：** 若 `file-encode` 专页内仍嵌入与 `text-encode` 相同的工作台，则 Tab 条 + 上述「file」 panel 可并存；若未来拆为**无 Tab 的专页**，则本结构收缩为**单流：文件进 → 串出**，其它能力用页脚或相关工具链接到 `text-encode` / `text-decode`。

## 3. SEO 与关键词

**原则：** 与「文本 / 字符串 转 Base64」页**错开**表述，强调**文件、二进制、Data URL**（若产品支持）与**本地、在线工具**等自然组合。

| 项 | 建议方向 |
|----|----------|
| **标题** | 含「文件」「Base64」「编码（在线）」等，加站点名；避免与文本页同句式仅换一两个字。 |
| **描述** | 一两句说清：本地选文件转 Base64 或 Data URL、不上传、适合嵌入/调试用。 |
| **自然关键词（运营或 H 文案可覆盖）** | 文件转 base64、图片 base64、Data URL 生成、本地 base64 编码 等。 |
| **URL** | 保持 `…/file-encode` 等**稳定英文 slug**；多语言用现有 `hrefForLocale` 习惯即可。 |

## 4. 说明文案

| 层次 | 建议 | 放哪 |
|------|------|------|
| **短** | 本工具做什么、适合「要把小文件变成一串 Base64/Data URL」的场景；**不替代加密**。 | 标题下或主内容顶。 |
| **信任** | 数据在**浏览器内**处理、不上传（若属实，与现站一致）。 | 与隐私链一起或 workbench 首段。 |
| **略长（可选折叠）** | Data URL 含 `mime;base64,` 与「纯 Base64 一段」区别；**Base64 可逆**、别用来藏密码。 | 折叠或次要段落。 |
| **长篇教程** | 不强制；有帮助中心再链。 | — |
