# 在线工具集网站 — 项目需求文档（PRD）

> 文档用途：界定产品范围、用户与系统需求，作为设计与开发的验收依据。  
> 关联文档：`功能规划表.md`、`字体与UI设计文档.md`、`开发进度表.md`；各产品线细节见 `工具规划-*.md`。

---

## 1. 项目概述

### 1.1 产品定位

面向海外开发者与普通办公用户的 **纯前端在线工具集**：在浏览器内完成编解码、格式化、生成器、摘要/加密、图片处理、开发辅助类任务，**默认数据不上传服务器**。

### 1.2 目标用户

- 需要快速处理文本/JSON/Base64/哈希等的技术人员。  
- 需要压缩图片、转格式、简单 PDF 等非技术用户。  
- 主要使用场景：检索进入单工具页、收藏夹回访、移动端偶发使用。

### 1.3 商业与合规边界（首版）

- 首版以 **功能与体验** 为主，广告/付费策略另文定义时不与本 PRD 冲突即可。  
- 涉及 Cookie、Authorization、UA、密钥等页面须有 **隐私与安全提示**（见各工具规划文档原则）。

## 2. 范围说明

### 2.1 范围内（Must）

| 类别 | 说明 |
| --- | --- |
| 技术形态 | **纯前端**：核心计算与编解码在浏览器完成；站点可用 Next.js（App Router）部署为静态或边缘友好形态。 |
| 国际化 | 支持 **en、zh、ja、es、fr**；英文 **URL 无前缀**（内部 rewrite 到 `/en`），其余语言为 **`/{locale}` 前缀**；语言切换为下拉，使用 **`router.replace`**（客户端过渡、无整页刷新）。 |
| 工具能力 | 覆盖 `功能规划表.md` 中 **P0 / P1** 条目；P2 及以后按迭代拉取。 |
| 可访问性 | 基础键盘可操作、对比度满足常见 WCAG AA 目标（详见 UI 文档）。 |

### 2.2 范围外（首版不做）

| 项 | 说明 |
| --- | --- |
| 按 IP 自动切换语言 | 明确不纳入首版；用户用手动语言切换即可。 |
| 用户账号与云端同步 | 不要求登录、不要求跨设备同步历史输入。 |
| 真实 API 代理、公网 Webhook 收件、可配置 Mock 端点 | 受 CORS/入站限制，不归入「纯前端核心承诺」；若未来单独立项需新 PRD。 |
| 服务端代理解析用户 JSON Schema `$ref` | 首版若不做服务端，则不在首版承诺远程 `$ref` 拉取。 |

## 3. 功能需求摘要

详细条目、优先级与阶段划分见 **`功能规划表.md`**。逻辑分组与产品线说明见 `docs/工具规划-*.md`。

**首版必须交付的最小集合（摘要）**：

- 站点壳：首页/分类导航、工具详情页模板、语言切换、页脚隐私与「本地处理」声明。  
- 至少一条完整产品线达到可发布质量（建议：**Base64 工作台** 或 **JSON 工作台** 之一作为首发标杆）。  
- 其余工具按 `开发进度表.md` 分阶段上线，单页需满足：输入/输出、错误提示、大输入降级或上限说明（如适用）。

## 4. 非功能需求

### 4.1 性能

- 首屏关键路径可控：避免单页加载过大 WASM/未分包资源阻塞交互。  
- 大文本/大图片：主线程解析须有 **体积上限** 或 **Web Worker** 策略（与图片线、JSON 线规划一致）。

### 4.2 安全与隐私

- 不在 URL query 中默认携带密钥、密码、敏感 payload。  
- 若存在将结构化内容渲染为 DOM 的场景，须防 XSS（文本优先或经净化的策略）。  
- 第三方脚本与埋点：若引入，须在隐私政策中披露；首版可极简。

### 4.3 SEO 与可发现性

- 每个对外工具页独立 URL（slug）、可索引标题与描述；避免大量无差异薄页。  
- `robots.txt` / `sitemap.xml` 策略与部署环境一致即可。

### 4.4 兼容与环境

- 目标浏览器：**最近两个大版本的 Chrome、Firefox、Safari、Edge**（实现阶段可再收紧）。  
- 移动端：布局自适应；复杂编辑器类工具允许「桌面体验更佳」提示。

## 5. 技术约束（与实现对齐）

| 项 | 要求 |
| --- | --- |
| 框架 | **Next.js 16+**（以团队锁定的具体次版本为准），App Router。 |
| 语言 | TypeScript 为主。 |
| 部署 | 与纯前端一致：静态导出或支持边缘的中间件均可，但不依赖自有后端存储用户数据。 |

---

## 6. 验收原则（全局）

- 每个工具页：具备清晰 **用途说明**、**限制说明**、**本地处理声明**（模板可由 UI 文档规定）。  
- 语言切换：在 en/zh 间切换后，当前页关键 UI 文案一致可懂；URL 策略不破坏刷新与分享。  
- 无阻断级 Lint/构建错误；核心路径可手动验收通过。
